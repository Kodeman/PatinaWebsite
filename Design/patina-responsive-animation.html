Page Transitions

Recommendation: Use Framer Motion or Next.js page transitions for smooth navigation. Implement with care to not impact performance.

PhaseDurationAnimation
Exit 200ms Fade out, slight scale down (0.98)
Enter 300ms Fade in, slide up (10px)
Stagger 50ms delay per element Content appears top-to-bottom

Framer Motion Example

const pageVariants = {
  initial: { opacity: 0, y: 10 },
  enter: { 
    opacity: 1, 
    y: 0,
    transition: { duration: 0.3, ease: [0.4, 0, 0.2, 1] }
  },
  exit: { 
    opacity: 0, 
    scale: 0.98,
    transition: { duration: 0.2 }
  }
};

const staggerContainer = {
  enter: {
    transition: { staggerChildren: 0.05, delayChildren: 0.1 }
  }
};

Intersection Observer Pattern

const observerOptions = {
  threshold: 0.2,
  rootMargin: '0px 0px -10% 0px'
};

const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('animate-in');
      observer.unobserve(entry.target);
    }
  });
}, observerOptions);

Reduced Motion

Critical: Always respect prefers-reduced-motion for accessibility. Disable all non-essential animations.

@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}

/* React/JS check */
const prefersReducedMotion = 
  window.matchMedia('(prefers-reduced-motion: reduce)').matches;

Performance Guidelines

RuleWhy
Only animate transform and opacity GPU-accelerated, no layout recalculation
Use will-change sparingly Apply only to elements about to animate
Avoid animating during scroll Can cause jank on mobile
Keep animations under 300ms for interactions Feels responsive without delay
Stagger list animations More natural, less overwhelming
Test on low-end devices Animations can be choppy on older hardware
